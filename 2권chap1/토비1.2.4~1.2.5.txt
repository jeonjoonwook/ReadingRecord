1.2.4
프로퍼티 값 설정 방법

DI 통해 빈에 주입되는 것은 레퍼런스와 단순 값이다.
스프링에서 말하는 값은 스프링이 관리하는 빈이 아닌 모든 것

싱글톤은 동시성 문제 때문에 필드값을 함부로 수정하지 않음
대개는 필드에 있는 값은 읽기 전용인 경우가 대부분
값 설정할 때 초기값이라고 보면됨

xml은 <property> 태그 써서 설정
String이라면 가장 간단한데 다른 타입이라면 변환해 주면 됨

빈이 사용해야 할 값을 코드에 담지 않고 설정을 통해 런타임시에 주입하는 이유는 무엇?
  환경에 따라 매번 달라질 수 있는 값,  ex)DataSource의 DriverClass, URL, UserName
   - 소스 코드 수정 없이 지정해주기 위해
  => 코드와 외부 설정을 분리해서 얻을 수 있는 가장 큰 장점은 설정이 바뀌더라도 소스코드를 다시 컴파일 안해도 됨

@Value를 이용하면 애노테이션을 이용해서도 프로퍼티 값 지정할 수 있다.
   - @Value로 값을 설정해준다는 건 자바 코드와 컨테이너가 런타임 시에 주입하는 정보를 분리하겠다는 의미
   - 잘 사용되지는 않음

PropertyEditor와 ConversionService
 - String 이외의 타입을 변환할때 사용됨

환경에 따라 자주 바뀔 수 있는 내용은 프로퍼티 파일로 분리하는 것이 가장 깔끔
프로퍼티 파일에서 설정값을 분리하면 @Value를 효과적으로 쓸 수 있다. 
@Value에서 프로퍼티 파일의 내용을 참조하게 해주면 소스코드 수정 없이 Value를 통해 프로퍼티에 주입되는 값을 변경할 수 있음
- 능동 변환 또는 수동 변환을 통해 프로퍼티에 있는 값들을 value 애트리뷰트에 넣어 준다.

1.2.5
컨테이너가 자동 등록하는 빈

ApplicationContext, BeanFactory
-스프링에서는 컨테이너 자신을 빈으로 등록해두고 필요하면 일반 빈에서 DI 받아서 사용할 수 있음
-ApplicationContext는 BeanFactory를 상속 받은 것이므로 BeanFactory 메소드는 전부 ApplicationContext에서 지원됨
-ApplicationContext를 @autowired로 주입받아서 getBean 사용하자
-applicationContext는 Resourceloader를 상속하므로 getResource로 리소스를 읽어올수 있다.
